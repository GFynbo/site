Contract Database

PHP, MySQL, HTML, CSS, and JavaScript

What?

This is a web application that allows a business owner to have a personalized online contract for his customers to fill out and submit also allowing them to electronically sign said contracts. After the contract has been submitted the owner can then use multiple options on the database page to either send an email reminding them they have a payment due, print their contract out, or update their contact information.

How?

The Contract Database web application uses a combination of PHP (for MySQL handling and security features), MySQL (for data storage and efficiency), HTML, CSS, & JavaScript (for design and form handling). Initally the website prompts the user for a username and password via HTTP authentication on a private network. The website then takes a structured form, submits the information to the MySQL database, then directs the user to the database page. Utilizing PHP and MySQL the Customer Database page allows the user to see a formatted table of all the users with options to edit, print, or email each customer contract.

  • Edit — allows the user to see the information and edit it
  • Print — allows the user to print a minimalized form-like page
  • Send Contract — emails the customer contract to the email address in the contract, prompting for an electronic signature

MiniGoogle

Java

What?

MiniGoogle is a project that first started out as a small class project, but really took off when I became engrossed in the potential of the application. Ultimately, time caught up on me and I was no longer able to update it.

How?

MiniGoogle is a Java application that takes as many text files as necessary filled with a title and first paragraph of the related Wikipedia article and implements that into a hash table. It then uses a cosine similarity calculation to determine the relevancy of your search phrase to the articles. Finally, it displays the articles in descending order based on that similarity calculation.

Sim (Pencil Game)

Java

What?

Sim is defined by Wikipedia as

The game of Sim is played by two players on a board consisting of six dots ('vertices'). Each dot is connected to every other dot by a line ('edge'). Two players take turns coloring any uncolored lines. One player colors in one color, and the other colors in another color, with each player trying to avoid the creation of a triangle made solely of their color (only triangles with the dots as corners count; intersections of lines are not relevant); the player who completes such a triangle loses immediately.

which are the same rules that apply to this Java version of the game.

How?

Using an evaluation algorithm for each move, the game would test thousands of different moves. Using a tree data structure, the program kills certain paths of moves thanks to the algorithm concluding that given a specific move, then no further moves could be better. Choosing the most logical choice based on scoring specific strategies and efficient tree structures, the game would move in real time with a difficulty that causes nearly all inexperience players to lose every game.
